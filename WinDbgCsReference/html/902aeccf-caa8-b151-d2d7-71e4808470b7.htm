CacheUserTypeFields
Gets or sets a value indicating whether generated user type should cache value of fields. While using this will result in better performance when calling properties multiple times, it will also drastically impact user type creation (specially when user type has a lot of fields).
CommonTypesNamespace
Gets or sets the namespace name for types found in multiple modules.

Gets or sets a value indicating whether generated code should be compressed. Compressed code has one space indent, removes comments and uses single like for generating properties.

ForceUserTypesToNewInsteadOfCasting
Gets or sets a value indicating whether generated code should use operator new for creating user types instead of casting.
GenerateAssemblyWithRoslyn
Gets or sets a value indicating whether generated assembly should be compiled with Roslyn.
GeneratedAssemblyName
Gets or sets the name of the generated assembly. If not empty, code generator will compile generated code into generated assembly.
GeneratedPropsFileName
Gets or sets the name of the generated props file. If not empty, code generator will generate props file that contains compile statement for each generated code file.
GenerateNamespaceAsStaticClass
Gets or sets a value indicating whether namespace should be generated as static class.
GeneratePhysicalMappingOfUserTypes
Gets or sets a value indicating whether generated code will use physical mapping instead of symbolic to access fields. Using this greatly improve performance, but lower re-usability of generated code after adding new fields. When using this, consider making code generator part of the build.
IncludedFiles
Gets or sets the list of files to be compiled when generating assembly. Since all exported user types are partial classes, you can easily add code for it. For example, if you have container class, you can add include file to implement IEnumerable interface for that type.
LazyCacheUserTypeFields
Gets or sets a value indicating whether generated user type should lazily cache value of fields. While using this will result in better performance when calling properties multiple times, it will also drastically impact user type memory usage and somewhat impact its creation.
Modules
Gets or sets the list of modules for which user types should be exported.
MultiFileExport
Gets or sets a value indicating whether generated code should be saved in multiple files. If set to true, every generated class/enum will be saved in separate file and every namespace will be separate folder.
MultiLineProperties
Gets or sets a value indicating whether generated properties should be formatted in multiple lines.
ReferencedAssemblies
Gets or sets the list of referenced assemblies when generating assembly.
Transformations
Gets or sets the list of transformations that will be applied on generated user types.
Types
Gets or sets the list of user types to be exported. If the list is empty, all user type will be exported.
UseDiaSymbolProvider
Gets or sets a value indicating whether generated code will use DIA symbol provider.
UseHungarianNotation
Gets or sets a value indicating whether generator should try to match use of Hungarian notation.
